# OSSM v0.1.2 Release Plan

## Overview

Prepare OSSM for public release with end-user focused documentation, a self-documenting case sticker, and QA verification.

**Design Principle**: "10-year self-documenting" - someone finding this device in a decade should be able to understand and troubleshoot it without external resources.

## 3. Sticker Design

**Dimensions**: 95mm x 85mm (fits Deutsch PCB case flat surface)

**Format**: SVG (flexible for printing adjustments)

### Layout

```
┌─────────────────────────────────────────────────────┐
│  OSSM v0.1.2          [QR CODE]                     │
│  Open Source Sensor Module    → github.com/...      │
├─────────────────────────────────────────────────────┤
│  A           B           C           D              │
│ 1.GND       1.temp3-    1.temp5+    1.SCL           │
│ 2.pres2     2.temp3     2.pres3+    2.SDA           │
│ ...         ...         ...         ...             │
│12.temp1-   12.pres6    12.pres4+   12.temp8-        │
├─────────────────────────────────────────────────────┤
│ 12V Power (A7) │ CAN: D7(L) D8(H) │ J1939 @250kbps │
│ github.com/jlaustill/ossm                           │
└─────────────────────────────────────────────────────┘
```

### Design Requirements

- High contrast (black on white) for engine bay readability
- 6-7pt font for pin table
- QR code ~15mm for phone scanning at arm's length
- Weatherproof printing recommended (vinyl or laminated)

### Content

**Header**: Project name, version, QR code to GitHub repo

**Body**: 4-column pinout table (Connectors A, B, C, D with 12 pins each)

**Footer**: Power requirements, CAN pinout, protocol info, GitHub attribution

## 4. QA Testing

Manual verification of all hardware and software functionality before release.

### Hardware Sensors
- [ ] temp1-temp8: Each NTC reads reasonable values
- [ ] pres1-pres7: Each pressure sensor reads within range
- [ ] EGT (MAX31856): Thermocouple reads, fault detection works
- [ ] BME280: Ambient temp, humidity, barometric all reporting

### Configuration
- [ ] Enable/disable SPNs via J1939 (PGN 65280)
- [ ] Enable/disable SPNs via Serial
- [ ] Config persists across power cycle (EEPROM)
- [ ] Factory reset works (command 7)

### J1939 Output
- [ ] All PGNs transmit at correct intervals
- [ ] Only enabled SPNs are transmitted
- [ ] Values match sensor readings (scaling/offsets correct)

### Edge Cases
- [ ] Disconnected sensor shows fault, not garbage data
- [ ] Out-of-range values handled gracefully
- [ ] Multiple sensors enabled simultaneously

## 5. Release Process

1. Complete all documentation tasks
2. Create sticker SVG
3. Manual QA testing
4. Create git tag `v0.1.2`
5. Create GitHub Release:
   - Release notes from CHANGELOG
   - Pre-compiled `.hex` firmware binary
   - Link to hardware PDFs
